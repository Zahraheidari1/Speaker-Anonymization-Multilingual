# Speaker Anonymization Pipeline

A complete speaker anonymization workflow built on top of the [IMS Speaker Anonymization](https://github.com/DigitalPhonetics/speaker-anonymization) system. The pipeline covers data preprocessing, ASV model training, evaluation, and comparison — all runnable from a single notebook.

---

## System Description

The anonymization system operates in three stages:

1. **Feature Extraction** — Three information streams are extracted from the input signal:
   - Linguistic content via an end-to-end ASR model (represented as text transcriptions using Whisper-large-v3)
   - Prosody: phone-level normalized pitch, energy, and duration
   - Speaker identity: GST-style speaker embeddings

2. **Anonymization** — The speaker embedding is replaced with an artificial one generated by a GAN. Cosine distance is used to ensure the new embedding is sufficiently different from the original.

3. **Synthesis** — A FastSpeech2-based TTS and HiFiGAN vocoder re-synthesize the speech using the anonymized speaker embedding while preserving linguistic content and prosody.

## Pipeline Overview

The notebook `speaker_anonymization_pipeline.ipynb` runs the full experiment in five steps:

| Step | Description |
|------|-------------|
| **0 — Preprocess** | Scans LibriSpeech, splits speakers into enroll/trial sets, generates Kaldi-format files and CSVs |
| **1 — Train Baseline** | Trains an ECAPA-TDNN ASV model with default hyperparameters |
| **2 — Evaluate Baseline** | Runs ASV and ASR evaluation on original and anonymized speech |
| **3 — Train Improved** | Trains with optimized hyperparameters (higher AAM margin, faster LR cycling) |
| **4 — Evaluate Improved** | Evaluates the improved model |
| **5 — Compare** | Side-by-side comparison of baseline vs improved EER and WER |

## Configuration

Key parameters set at the top of the notebook:

```python
LIBRISPEECH_PATH  = '../data/dev-clean/LibriSpeech/dev-clean'
TRAIN_SPK_RATIO   = 0.7     # 70% speakers → enrollment/train
CSV_TRAIN_RATIO   = 0.8     # 80% of enroll speakers → train, 20% → dev
RANDOM_SEED       = 500
SAMPLE_RATE       = 16000
MIN_DURATION      = 3.0     # seconds — utterances shorter than this are skipped
```

## Results

### Baseline vs Improved

| Feature | Baseline | Improved |
|---------|----------|----------|
| Architecture | ECAPA-TDNN 192-dim | ECAPA-TDNN 192-dim |
| Pooling | Attentive Statistics | Attentive Statistics |
| AAM Margin | 0.2 | **0.3** |
| LR Step Size | 65,000 | **2,000** |
| Weight Decay | 2e-6 | **1e-5** |
| **ASV EER** | 11.667% | **13.333% ↑** |
| **ASR WER (original)** | 10.596% | 10.596% |
| **ASR WER (anonymized)** | 19.478% | 19.478% |

> Higher EER = harder to identify the speaker = better privacy.

### Key Observations

- **Privacy improvement:** Increasing the AAM margin to 0.3 pushes the model to learn more distinct speaker boundaries, raising EER by +1.67 percentage points.
- **Effective LR scheduling:** Reducing the step size to 2,000 allows the cyclic learning rate to actually complete cycles within the 100-epoch window, improving fine-tuning stability.
- **No utility loss:** WER remains constant at 19.478% on anonymized speech — the improved privacy comes at no cost to speech intelligibility.

## Data

Experiments use the **LibriSpeech dev-clean** subset (40 speakers, 2,703 utterances, ~5.4 hours).

Speaker split:
- **70%** of speakers → enrollment set (used for ASV training)
- **30%** of speakers → trial set (evaluation only, never seen during training)

## Running the Pipeline

Open and run `speaker_anonymization_pipeline.ipynb` top to bottom. Each step is self-contained and labeled. GPU IDs are passed as arguments to the underlying evaluation scripts.

For anonymization and evaluation scripts directly:

```bash
# Anonymization
python run_anonymization.py --config anon/anon_ims_sttts_pc_whisper.yaml --lang en --gpu_ids 0

# Evaluation (pre-trained model)
python run_evaluation.py --config eval_pre/eval_pre_whisper.yaml --lang en --gpu_ids 0
```
